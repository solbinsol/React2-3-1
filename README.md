 ## 커스텀 서버가 필요한 경우
 - 실제 웹 앱을 만들고 서비스를 제공할때 Express.js 나 Fastify 서버에서 앱을 실행하는 경우는 거의 없음
 - 다만 일부 환경에서는 반드시 커스텀 서버가 필요할 수 있음.
 ### 어떨때 사용하나 
 - 1. 웹 앱을 기존의 서버에서 실행하는 경우
  - 기존 서버에서 실행되던 로직이나 미들웨어 등을 최대한 유지하고 싶을때
 - 2. 멀티테넌지 지원이 필요한 경우
  - Next.js는 멀티 도메인 지원, 하지만 도메인수가 너무 많아지면 커스텀 서버의 멀티테넌트를 사용하는 것이 효울적
- 3. 더 세밀한 제어가 필요할 경우
 - 웹앱의 복잡도가 증가해서 앱의 백엔드 코드를 다른 방법으로 관리하고 싶을 때

- 클라이언트 사이드 렌더링만 지원

## 왜 Next.js 일까 ?
- React 에서 제공하지 않는 여러 기능 지원
- 설정이나 개발 옵션 등 다양한 부분에서도 유용한 기능들을 지원
- 활동적인 커뮤니티 있어 개발 단계별로 많은 지원 받을 수 있음
-  ### 서버사이드 렌더링 !
